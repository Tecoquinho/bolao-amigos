Regras obrigatórias:

* Código simples é melhor que código inteligente.
* Evitar overengineering.
* Evitar abstrações desnecessárias.
* Seguir SOLID quando fizer sentido.
* Seguir Clean Code.
* Explicar decisões arquiteturais importantes.
* Nunca gerar código duplicado.
* Priorizar legibilidade.
* Priorizar manutenção futura.

Sempre que iniciar uma tarefa:

1. Entenda o requisito.
2. Explique rapidamente a solução.
3. Liste arquivos afetados.
4. Só então implemente.
